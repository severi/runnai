How it works

RunnAI is a coaching agent that runs in your terminal. It syncs with your Strava account, builds a picture of who you are as a runner, and uses that context every time you talk to it. Unlike a regular chatbot, it maintains structured memory across sessions.

  • Persistent memory Three tiers: a hot cache loaded into every conversation, a deep memory store for detailed observations, and a SQLite database with all your activity data. The agent decides what to remember, promote, or archive.
  • Strava integration Syncs your activities, tracks best efforts, and can write AI-generated analysis back to your Strava activity descriptions. Full OAuth flow, all data stays on your machine.
  • Adaptive training plans Periodized plans that adjust weekly based on what actually happened — not a static spreadsheet. Accounts for fatigue, missed sessions, and how your body responds to load.
  • Coaching skills Modular knowledge for periodization, injury management, workout analysis, and race prediction, grounded in running science rather than generic LLM output.

Getting started

Prerequisites

  • Bun runtime
  • Strava API application — create one at strava.com/settings/api, set callback domain to localhost. This gives you a Client ID and Client Secret.
  • Claude auth — either an Anthropic API key or Claude Code installed (Pro/Max subscription — RunnAI piggybacks on its auth)

Install and configure

$ git clone https://github.com/severi/runnai.git
$ cd runnai && bun install
$ cp .env.example .env

Edit .env and fill in:

  • STRAVA_CLIENT_ID and STRAVA_CLIENT_SECRET from your Strava app
  • ANTHROPIC_API_KEY if using an API key (skip if using Claude Code auth)

Run

$ bun run build && bun run start

On first launch the coach connects your Strava account via OAuth, syncs your activities, and builds your profile.
